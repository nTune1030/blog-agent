================================================================================
          API CREDENTIALS SETUP GUIDE — Step-by-Step Walkthrough
          Project: blog-agent · Author: NTUNE1030
================================================================================

This guide walks you through setting up API credentials for all three
platforms so the Promotion Agent can post automatically.

================================================================================
PLATFORM 1: REDDIT (via PRAW)
================================================================================

PREREQUISITE: You need a Reddit account.

STEP 1: Create a Reddit App
  1. Log in to Reddit at https://www.reddit.com/
  2. Go to https://www.reddit.com/prefs/apps
  3. Scroll down and click "create another app..." (or "create an app")
  4. Fill in the form:
     - name:           PromotionAgent
     - App type:        Select "script" (NOT "web app")
     - description:     Agent for posting project promotions
     - about url:       (leave blank)
     - redirect uri:    http://localhost:8080  (required but unused for script apps)
  5. Click "create app"

STEP 2: Copy Your Credentials
  After creating the app, you'll see:
  
  ┌─────────────────────────────────────────────────┐
  │  PromotionAgent                                  │
  │  personal use script                             │
  │                                                   │
  │  client_id:     abc123XYZ_defghi     ← UNDER the │
  │                                  app name, small  │
  │  secret:       AbCdEf123456GhIjKl   ← Listed as  │
  │                                  "secret"         │
  └─────────────────────────────────────────────────┘

  - client_id     = the string directly under the app name
  - client_secret = the string labeled "secret"

STEP 3: Fill In promotion_profile.json
  Open promotion_profile.json and update api_credentials.reddit:

  "api_credentials": {
    "reddit": {
      "client_id": "PASTE_YOUR_CLIENT_ID_HERE",
      "client_secret": "PASTE_YOUR_CLIENT_SECRET_HERE",
      "username": "YOUR_REDDIT_USERNAME",
      "password": "YOUR_REDDIT_PASSWORD",
      "user_agent": "PromotionAgent/1.0 by YOUR_REDDIT_USERNAME"
    }
  }

  ⚠️  SECURITY: Your Reddit password is stored in plaintext. Consider
  using a Reddit account specifically for bot/automation purposes, not
  your personal main account.

================================================================================
PLATFORM 2: TWITTER/X (via Tweepy)
================================================================================

PREREQUISITE: You need a Twitter account and a phone number for
developer access.

STEP 1: Apply for a Developer Account
  1. Go to https://developer.twitter.com/en/portal/petition/essential-info
  2. Sign in with your Twitter account
  3. Fill out the application:
     - What is your primary reason for using Twitter APIs?
       → "Building tools to automate posting project updates"
     - Will you make your app content available to government entities?
       → Answer honestly (usually No)
  4. Agree to the Developer Agreement
  5. You may get instant approval, or it may take hours/days

STEP 2: Create a Project and App
  1. Go to https://developer.twitter.com/en/portal/dashboard
  2. Click "+ Create Project"
  3. Project name: PromotionAgent
  4. Use case: "Building tools to automate posting"
  5. Project description: "Automated project promotion agent"
  6. Click "+ Add App"
  7. App name: promotion-agent
  8. Environment: Development

STEP 3: Generate API Keys
  After creating the app, you'll see your Keys and Tokens:

  REQUIRED — API Keys (generate if not shown):
  - api_key            = "API Key" (also called Consumer Key)
  - api_secret         = "API Key Secret" (also called Consumer Secret)

  REQUIRED — Access Tokens (click "Generate" if not shown):
  - access_token       = "Access Token"
  - access_token_secret = "Access Token Secret"

  OPTIONAL — Bearer Token (for API v2 read-only):
  - bearer_token       = "Bearer Token"

STEP 4: Set App Permissions to READ AND WRITE
  ⚠️  CRITICAL: By default, apps are Read-Only. You MUST change this.

  1. Go to your app's "Settings" tab
  2. Scroll to "App permissions"
  3. Change from "Read" to "Read and Write"
  4. Save
  5. REGENERATE the Access Token and Access Token Secret
     (the old ones only have Read permission)

STEP 5: Fill In promotion_profile.json
  Open promotion_profile.json and update api_credentials.twitter:

  "api_credentials": {
    "twitter": {
      "api_key": "PASTE_YOUR_API_KEY",
      "api_secret": "PASTE_YOUR_API_SECRET",
      "access_token": "PASTE_YOUR_ACCESS_TOKEN",
      "access_token_secret": "PASTE_YOUR_ACCESS_TOKEN_SECRET",
      "bearer_token": "PASTE_YOUR_BEARER_TOKEN"
    }
  }

================================================================================
PLATFORM 3: DISCORD (via Webhooks)
================================================================================

PREREQUISITE: You need "Manage Webhooks" permission in the target
Discord server.

STEP 1: Create a Webhook
  1. Open Discord and go to your target server
  2. Right-click the channel where you want to post
     (e.g., #tools, #projects, #showcase)
  3. Click "Edit Channel"
  4. Go to "Integrations" → "Webhooks"
  5. Click "New Webhook"
  6. Name it: Promotion Agent
  7. Choose an avatar (optional)
  8. Click "Copy Webhook URL"
  9. Click "Save Changes"

STEP 2: Fill In promotion_profile.json
  Open promotion_profile.json and update api_credentials.discord:

  "api_credentials": {
    "discord": {
      "webhook_urls": [
        "https://discord.com/api/webhooks/123456789/AbCdEfGhIjKlMnOpQrStUvWxYz"
      ]
    }
  }

  To post to multiple channels, add multiple webhook URLs:
  "webhook_urls": [
    "https://discord.com/api/webhooks/CHANNEL_1_WEBHOOK",
    "https://discord.com/api/webhooks/CHANNEL_2_WEBHOOK"
  ]

================================================================================
AFTER SETUP: Verify Everything Works
================================================================================

1. Check readiness:
   python promote.py --readiness

   You should see:
   Reddit:    READY TO POST ✅
   Hacker News: MANUAL ONLY
   Discord:   READY TO POST ✅
   Twitter/X: READY TO POST ✅

2. Dry-run posting (no actual posts):
   python promote.py --post

3. Actually post (requires YES confirmation per platform):
   python promote.py --post --confirm

================================================================================
SECURITY NOTES
================================================================================

⚠️  promotion_profile.json contains plaintext API secrets and passwords.
    ADD IT TO .gitignore — never commit this file to a public repo!

⚠️  Reddit: Consider using a dedicated bot account, not your personal one.
    Reddit may shadowban accounts that post too frequently or appear automated.

⚠️  Twitter: Respect rate limits. Free tier allows 1,500 tweets/month.
    Do not spam — post once per project launch, not repeatedly.

⚠️  Discord: Webhook URLs are like passwords — anyone with the URL can post.
    Keep them private and rotate if leaked.
